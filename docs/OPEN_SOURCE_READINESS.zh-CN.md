# 开源准备情况

## 当前项目概况

`CDriveCleaner` 当前是一个基于 Rust + Tauri + React 的桌面应用，重点面向 Windows 磁盘清理与全盘扫描分析。

### 状态快照

记录时间：2026-04-03

- 仓库状态：当前目录已完成 git 初始化，`.gitignore` 已到位，同时已经补齐 `LICENSE`、`CHANGELOG.md`、`CONTRIBUTING*`、`CODE_OF_CONDUCT*`、`SECURITY*`、`rust-toolchain.toml`、`CODEOWNERS` 以及基础 GitHub Actions CI / release 工作流等公开仓库基础文件。
- 公开文档状态：根目录 `README.md` 与 `README.zh-CN.md` 已经从内部工作区说明升级为中英双语公开入口。
- 产品状态：桌面端已经具备 Home、Results、Categories、History、Settings，以及全盘扫描分析、导出流程、受限计划扫描和大树性能优化等主能力。
- 质量状态：当前工作区已经具备桌面端单测/集成测试、Windows CI 中的端到端 UI 自测、Rust 工作区测试，以及带比较和聚合能力的全盘扫描压力脚本。
- 开源准备状态：仓库已经跨过基础文档与治理文件门槛，但公开发布打包链路，以及贡献/安全流程的进一步细化仍待完成。

### 当前重点

- 持续加固大规模全盘扫描树的性能与体验。
- 继续缩小 CLI 与桌面端能力差距。
- 完善安装包、签名与公开发布流程。
- 让 README、开源准备文档和基准工具始终与实际产品状态保持一致。

当前工作区结构如下：

- `apps/desktop/`：桌面端 UI、Tauri 桥接层、端到端自测脚本
- `crates/core/`：清理引擎、全盘扫描展开逻辑、计划任务、导出能力
- `crates/cli/`：验证工具、压力测试工具，CLI 能力仍在继续补齐
- `crates/contracts/`：共享数据契约与测试夹具
- `docs/`：产品、架构、实施计划与路线图文档
- `scripts/`：构建辅助脚本、回归工具、性能基准脚本

## 当前已经具备的基础

- 桌面端主流程已经成型：Home、Results、Categories、History、Settings 都已具备可运行实现。
- 回归链路比较完整：单测、集成测试、全 UI 自测、Rust 工作区测试、全盘扫描压力脚本都已经存在。
- 全盘扫描性能优化已经进入持续迭代状态：分支保留合并、增量索引、搜索优化、树窗口化、多轮基准统计都已开始落地。
- `docs/` 下已经有架构、路线图、实施计划等文档基础，适合继续整理成公开资料。

- 最终确定后的公开安全报告渠道。
- 随着外部协作增加而持续细化的社区协作预期。
- 更完整的 GIF / 演示视频等动态素材，以及进一步优化的 README 首屏展示效果。
- 持续审查并清理机器私有路径、内部备注、生成产物等不应进入公开仓库的内容。

## 推荐的开源准备分阶段方案

### 阶段 1：仓库卫生整理

- 只保留源码、文档、夹具与明确需要保留的基准数据。
- 忽略本地状态、生成产物、构建目录、依赖目录。
- 检查文档中是否仍包含本机绝对路径与内部工作流假设。

### 阶段 2：公开文档基线

- 持续完善根目录 `README.md` 的截图、支持范围和体验说明。
- 持续让贡献指南、行为准则、安全说明与实际协作流程保持一致。
- 以英文为主版本，同时提供中文补充文档。

### 阶段 3：发布准备

- 准备演示流程、GIF / 视频素材，以及更完整的支持范围说明。
- 明确当前稳定能力与仍在加固中的边界。

### 阶段 4：社区协作准备

- 写清本地开发命令。
- 写清 PR 前需要通过的测试。
- 定义 issue 标签、roadmap 归类与维护边界。
